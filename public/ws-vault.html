🤖 Claude AI — How It Integrates P&P + BBP

Claude (Anthropic) is embedded into EVL in two ways — as a build partner and as a live AI engine on the Compare page.

RoleHow It WorksEVL Application
🔧 Build Partner Claude writes, edits, and deploys all code directly to GitHub → Vercel via API token. No developer needed. Every page on the site was built this way — HTML, CSS, JS, API routes, all of it.
📋 P&P Integration Policies & Procedures are fed into Claude as context in each session. Claude applies EVL rules (no delivery without payment, 72hr timeline, deposit structure) automatically. Every customer flow, authorization language, and disclosure was written by Claude using EVL P&P as the foundation.
💡 BBP / BITOP Best Business Practices and Best In Theory, Optimal Practice — Claude references the EVL journal and blueprint to keep all builds aligned with the business model. Pricing logic, deal sequencing, trade card structure, WS exchange flow — all built to BBP spec.
⚡ Live AI Engine The Compare page calls Claude Sonnet via api/compare.js — a Vercel serverless function — in real time. Vehicle comparisons, trim breakdowns, cost of ownership — all powered by Claude on demand.
📖 Session Memory Claude retains EVL build history across sessions via the project memory system and past conversation search. No need to re-explain the business — Claude picks up exactly where we left off every session.
